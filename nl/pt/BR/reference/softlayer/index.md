# Comandos do {{site.data.keyword.BluSoftlayer_notm}} (bluemix sl)
{: #softlayer_cli}

O plug-in do {{site.data.keyword.BluSoftlayer}} foi mesclado na CLI do {{site.data.keyword.Bluemix_notm}}. Não é mais necessário instalar o plug-in.

### Uso do comando
Para visualizar informações da ajuda dos comandos, execute: `bluemix sl [command] -h`.

### bluemix sl init
{: #sl_init}

Inicialize as definições de configuração usadas para se conectar ao ambiente do SoftLayer. A configuração inclui o nome do usuário, a chave API ou a senha, a conta e o terminal.
```
bluemix sl init [OPTIONS]
```

<strong>Opções de comando</strong>:
   <dl>
   <dt>-a,--api-endpoint</dt>
   <dd>URL do terminal da API do Softlayer, o padrão é: https://api.softlayer.com/rest/v3.1</dd>
   <dt>-u,--sl-user</dt>
   <dd>Nome do usuário do Softlayer</dd>
   <dt>-p,--sl-password</dt>
   <dd>Senha do Softlayer ou chave API</dd>
   <dt>-c,--account-id</dt>
   <dd>ID da conta do Softlayer</dd>
   </dl>

Por exemplo, efetue login com o nome do usuário e a senha/chave API do Softlayer

### bluemix sl block access-authorize
{: #sl_block_access_authorize}

Autorizar os hosts a acessarem um determinado volume.
```
bluemix sl block access-authorize VOLUME_ID [OPTIONS]
```

<strong>Opções de comando</strong>:
   <dl>
   <dt>--hardware-id</dt>
   <dd>O ID de um servidor de hardware a ser autorizado.</dd>
   <dt>--virtual-id</dt>
   <dd>O ID de um servidor virtual a ser autorizado.</dd>
   <dt>--ip-address-id</dt>
   <dd>O ID de um endereço IP a ser autorizado.</dd>
   <dt>--ip-address</dt>
   <dd>Um endereço IP a ser autorizado.</dd>
    </dl>

**Exemplos**:
```
bluemix sl block access-authorize 12345678 --virtual-id 87654321
```
Esse comando autoriza o servidor virtual com ID `87654321` a acessar o volume com ID `12345678`.

### bluemix sl block access-list
{: #sl_block_access_list}

Listar ACLs.
```
bluemix sl block access-list VOLUME_ID [OPTIONS]
```
<strong>Opções de comando</strong>:
   <dl>
   <dt>--sortby</dt>
   <dd>Coluna pela qual classificar, as opções são: `id`, `name`,  `type`, `private_ip_address`, `host_iqn`, `username` ou `password`.</dd>
   <dt>--columns</dt>
   <dd>  Colunas para exibição, as opções são: `id`, `name`, `type`, `private_ip_address`, `host_iqn`, `username` ou `password`.</dd>
</dl>

**Exemplos**:
```
bluemix sl block access-list 12345678 --sortby id
```
Esse comando lista todos os hosts que têm autorização para acessar o volume com ID `12345678` e os classifica pelo ID.

### bluemix sl block access-revoke
{: #sl_block_access_revoke}

Revogar a autorização de hosts que acessam um determinado volume.
```
 bluemix sl block access-revoke VOLUME_ID [OPTIONS]
```
<strong>Opções de comando</strong>:
   <dl>
   <dt>--hardware-id</dt>
   <dd>O ID de um servidor de hardware a ser autorizado.</dd>
   <dt>--virtual-id</dt>
   <dd>O ID de um servidor virtual a ser autorizado.</dd>
   <dt>--ip-address-id</dt>
   <dd>O ID de um endereço IP a ser autorizado.</dd>
   <dt>--ip-address</dt>
   <dd>Um endereço IP a ser autorizado.</dd>
    </dl>

**Exemplos**:
```
bluemix sl block access-revoke 12345678 --virtual-id 87654321
```
Esse comando revoga o acesso de servidor virtual com ID `87654321` para o volume com ID `12345678`.

### bluemix sl block replica-failback
{: #sl_block_replica_failback}

Causar failback de um volume de bloco por meio da réplica.
```
 bluemix sl block replica-failback VOLUME_ID
```
**Exemplos**:
```
 bluemix sl block replica-failback 12345678
```
Esse comando executa a operação de failback para o volume com ID `12345678`.

### bluemix sl block replica-failover
{: #sl_block_replica_failover}

Causar failover de um volume de bloco no volume de réplica especificado.
```
bluemix sl block replica-failover VOLUME_ID REPLICA_ID
```

**Exemplos**:
```
 bluemix sl block replica-failover 12345678 87654321
```
Esse comando executa a operação de failover do volume com ID `12345678` para o volume de réplica com ID `87654321`.

### bluemix sl block replica-order
{: #sl_block_replica_order}

Pedir um volume de réplica de armazenamento de bloco.
```
 bluemix sl block replica-order VOLUME_ID [OPTIONS]
```
<strong>Opções de comando</strong>:
   <dl>
   <dt>-s, --snapshot-schedule</dt>
   <dd>Requerido. Planejamento da captura instantânea a ser usado para replicação. As opções são: `HOURLY`, `DAILY` ou `WEEKLY`.</dd>
   <dt>-d, --datacenter</dt>
   <dd>Requerido. Nome abreviado do data center da réplica. Por exemplo, `dal09`.</dd>
   <dt>--tier</dt>
   <dd>Opcional. Camada de armazenamento de resistência (IOPS por GB) do volume primário para o qual uma réplica é pedida. As opções são: `0.25`, `2`, `4` ou `10`.</dd>
   <dt>--os-type</dt>
   <dd>Opcional. Tipo de sistema operacional do volume primário para o qual uma réplica é pedida. As opções são: `HYPER_V`, `LINUX`, `VMWARE`, `WINDOWS_2008`, `WINDOWS_GPT`, `WINDOWS` ou `XEN`.</dd>
   <dt>-f, --force</dt>
   <dd>Forçar a operação sem confirmação.</dd>
    </dl>

**Exemplos**:
```
bluemix sl block replica-order 12345678 -s DAILY -d dal09 --tier 4 --os-type LINUX
```
Esse comando pede uma réplica para o volume com ID `12345678`, que executa a replicação DIÁRIA, está localizado em dal09 com o nível de camada `4` e o OS tipo `Linux`.

### bluemix sl block snapshot-cancel
{: #sl_block_snapshot_cancel}

Cancelar o espaço de captura instantânea existente de um determinado volume.
```
 bluemix sl block snapshot-cancel SNAPSHOT_ID [OPTIONS]
```

<strong>Opções de comando</strong>:
   <dl>
   <dt>--reason</dt>
   <dd>Um motivo opcional para cancelamento.</dd>
   <dt>--immediate</dt>
   <dd>Cancelar o espaço de captura instantânea imediatamente, em vez de no aniversário de faturamento.</dd>
   <dt>-f, --force</dt>
   <dd>Forçar a operação sem confirmação.</dd>
	</dl>

**Exemplos**:
```
 bluemix sl block snapshot-cancel 12345678 --immediate -f
```

### bluemix sl vs create
{: #sl_vs_create}

Criar instância do servidor virtual
```
   bluemix sl vs create [OPTIONS]
```

<strong>Opções de comando</strong>:
   <dl>
   <dt>-H, --hostname</dt>
   <dd>Parte de host do FQDN [necessário]</dd>
   <dt>-D, --domain</dt>
   <dd>Parte de domínio do FQDN [necessário]</dd>
   <dt>-c, --cpu</dt>
   <dd>Número de núcleos da CPU [necessário]</dd>
   <dt>-m, --memory</dt>
   <dd>Memória em megabytes [necessário]</dd>
   <dt>-d, --datacenter</dt>
   <dd>Nome abreviado do data center [necessário]</dd>
   <dt>-o, --os</dt>
   <dd>Código de instalação do OS. Dica: é possível especificar <OS>_LATEST</dd>
   <dt>--image</dt>
   <dd>ID da imagem. Veja: [bluemix sl image list](#bluemix_sl_image_list) para obter referência</dd>
   <dt>--billing</dt>
   <dd>Taxa de faturamento. O padrão é: de hora em hora. As opções são: de hora em hora, mensal</dd>
   <dt>--dedicated</dt>
   <dd>Criar um Servidor virtual dedicado (Nó privado)</dd>
   <dt>--san</dt>
   <dd>Usar armazenamento SAN em vez de disco local</dd>
   <dt>--test</dt>
   <dd>Não criar realmente o servidor virtual</dd>
   <dt>--export</dt>
   <dd>Exporta opções para um arquivo de modelo</dd>
   <dt>-i, --postinstall</dt>
   <dd>Script de pós-instalação para download</dd>
   <dt>-k, --key</dt>
   <dd>Os IDs das chaves SSH a serem incluídas no usuário raiz (permitidas múltiplas ocorrências)</dd>
   <dt>--disk</dt>
   <dd>Tamanhos do disco (permitidas múltiplas ocorrências)</dd>
   <dt>--private</dt>
   <dd>Força o servidor virtual a ter acesso apenas à rede privada</dd>
   <dt>--like</dt>
   <dd>Usar a configuração de um servidor virtual existente</dd>
   <dt>-n, --network</dt>
   <dd>Velocidade da porta de rede em Mbps</dd>
   <dt>--tag</dt>
   <dd>Tags para incluir na instância (permitidas múltiplas ocorrências)</dd>
   <dt>-t, --template</dt>
   <dd>Um arquivo de modelo que padroniza as opções da linha de comandos</dd>
   <dt>-u, --userdata</dt>
   <dd>Sequência de metadados definida pelo usuário</dd>
   <dt>-F, --userfile</dt>
   <dd>Ler dados do usuário do arquivo</dd>
   <dt>--vlan-public</dt>
   <dd>O ID da VLAN pública na qual você deseja colocar o servidor virtual</dd>
   <dt>--vlan-private</dt>
   <dd>O ID da VLAN privada na qual você deseja colocar o servidor virtual</dd>
   <dt>--wait</dt>
   <dd>Aguarde até que o servidor virtual tenha concluído o fornecimento por até X segundos antes de retornar</dd>
   <dt>-f, --force</dt>
   <dd>Forçar operação sem confirmação</dd>
	</dl>

**Exemplos**:
```
  bluemix sl vs create -H myvsi -D bluemix.net -c 4 -m 4096 -d dal10 -o UBUNTU_16_64 --disk 100 --disk 1000 --vlan-public 413
```
Esse comando pede uma instância de servidor virtual com o nome de host myvsi, o domínio bluemix.net, 4 núcleos de CPU, 4.096 M de memória, localizada no data center: dal10,
o sistema de operação é UBUNTU 16 de 64 bits, 2 discos, um de 100 G, o outro de 1.000 G e colocado na vlan pública com ID 413.
```
  bluemix sl vs create -H myvsi -D bluemix.net -c 4 -m 4096 -d dal10 -o UBUNTU_16_64 --disk 100 --disk 1000 --vlan-public 413 --test
```
Esse comando testa se a ordem é válida com as opções acima antes de ela ser realmente feita.
```
bluemix sl vs create -H myvsi -D bluemix.net -c 4 -m 4096 -d dal10 -o UBUNTU_16_64 --disk 100 --disk 1000 --vlan-public 413 --export ~/myvis.txt
```

Esse comando exporta as opções acima para um arquivo: myvsi.txt sob o diretório inicial do usuário para uso posterior.

### bluemix sl vs upgrade
{: #sl_vs_upgrade}

Fazer upgrade de uma instância de servidor virtual
```
   bluemix sl vs upgrade IDENTIFIER [OPTIONS]
```
Nota: o SoftLayer reinicializa automaticamente a instância depois de a solicitação de upgrade ser feita. A instância é interrompida até a conclusão da transação de upgrade. No entanto, para a Rede, nenhuma reinicialização é necessária.

<strong>Opções de comando</strong>:
   <dl>
   <dt>-c, --cpu</dt>
   <dd>Número de Núcleos de CPU</dd>
   <dt>--private</dt>
   <dd>O núcleo da CPU ficará em um servidor host dedicado.</dd>
   <dt>-m, --memory</dt>
   <dd>Memória em megabytes.</dd>
   <dt>--network</dt>
   <dd>Velocidade da porta de rede em Mbps.</dd>
   <dt>-f, --force</dt>
   <dd>Forçar a operação sem confirmação.</dd>
    </dl>

**Exemplos**:
```
 bluemix sl vs upgrade 12345678 -c 8 -m 8192 --network 1000
```
Esse comando faz upgrade da instância de servidor virtual com ID 12345678 e configura o número de núcleos da CPU como 8, a memória como 8.192 M, a velocidade da porta de rede como 1.000 Mbps.



### bluemix sl vlan create
{: #sl_vlan_create}

```
   bluemix sl vlan create [OPTIONS]
```

<strong>Opções de comando</strong>:
   <dl>
   <dt>-t, --vlan-type</dt>
   <dd>O tipo da VLAN, público ou privado</dd>
   <dt>-r, --router</dt>
   <dd>O nome do host do roteador</dd>
   <dt>-d, --datacenter</dt>
   <dd>O nome abreviado do data center</dd>
   <dt>-s, --size</dt>
   <dd>O tamanho das sub-redes, as opções são: 8 (5 IPs utilizáveis) ou 16 (13 IPs utilizáveis) ou 32 (29 IPs utilizáveis)</dd>
   <dt>-n, --name</dt>
   <dd>O nome da VLAN</dd>
   <dt>-f, --force</dt>
   <dd>Forçar operação sem confirmação</dd>
    </dl>

**Exemplos**:
```
 bluemix sl vlan create -t public -d dal09 -s 16 -n myvlan
```
Esse comando cria uma vlan pública localizada no data center dal09 com 16 endereços IP e o nome é myvlan.
```
 bluemix sl vlan create -r bcr01a.dal09 -s 16 -n myvlan
```
Esse comando cria uma vlan localizada no roteador bcr01a.dal09 com 16 endereços IP e o nome é myvlan.
